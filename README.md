# Applied Engineering

**Process and Skill Developed by Austin Johnnic**

Architecting, building, and deploying full systems rapidly with structured reasoning and AI-accelerated execution.

## What This Is

A full systems engineering skill for Claude that plans production-grade software through exhaustive questioning, suggestion generation, competitive research, gap analysis, and comprehensive project document generation. Works with Claude Code, Cursor, Windsurf, Lovable, Bolt, Replit Agent, and Codex.

**Motto: "Never assume — always research. Never assume — always ask."**

## The Process

```
Stage 1: Context Gathering
Stage 2: Tech Stack Evaluation (always suggest alternatives)
Stage 3: Exhaustive Questions (150-500+ across 25+ categories)
Stage 4: Exhaustive Suggestions (100-300+ with effort/impact)
Stage 5: Gap Analysis + Competitive Research + Risk Assessment
Stage 6: Final Walkthrough (risky assumptions, completeness audit)
Stage 7: Project Document Generation (15-22 documents)
Stage 8: Confidence Assessment
Stage 9: Tool-Specific Execution Prompt
```

## Installation


## Planning Modes

- **Standard Plan** (2-4 hours): 150+ questions, 100+ suggestions, 12-15 documents
- **Deep/Enterprise Plan** (6-8+ hours): 300+ questions, 200+ suggestions, 18-22 documents

## Key Features

- 25+ question categories with 50+ questions each
- Competitive research protocol
- Enterprise architect expansion protocol
- Complexity scoring with over-engineering detection
- Planning checkpoints every 30 minutes
- MVP gate for scope management
- 15-22 project documents generated
- Stack-agnostic (supports all modern tech stacks)
- Tool-agnostic (Claude Code, Cursor, Lovable, etc.)
- Zero mock data policy
- Component library research and recommendations
