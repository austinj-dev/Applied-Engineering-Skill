# Suggestion Bank — Applied Engineering

## How This Bank Works
Mirrors the question bank structure. 25+ categories, 50+ suggestions each.
Every suggestion includes: description, rationale, effort, impact, dependencies.

## Suggestion Format
```
**[N].** [Description]
- WHY: [rationale]
- EFFORT: [hours/days]
- IMPACT: [Critical/High/Medium/Low]
- DEPS: [dependencies]
- CATEGORY: Include Now / Defer / Reject
```

## Suggestion Waves
- Wave 1: Core architecture + infrastructure
- Wave 2: Feature enhancements + patterns
- Wave 3: Enterprise patterns (triggers enterprise-architect-protocol.md)
- Wave 4: Future-proofing + competitive gaps

## Categories (match question bank)
1. Product & Business Model suggestions
2. Tech Stack suggestions (with alternatives and trade-offs)
3. Database Architecture suggestions
4. Auth & Security suggestions
5-30. [One category per question bank category]

## Auto-Generated Suggestions
These generate for EVERY project regardless:
- OWASP security hardening (per Top 10 2025)
- Infrastructure baseline (monitoring, logging, health checks)
- Developer experience (local dev, test data, documentation)
- Performance baseline (lazy loading, code splitting, image opt)
- Accessibility baseline (WCAG AA compliance path)
