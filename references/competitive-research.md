# Competitive Research Protocol — Applied Engineering

## When to Activate
When user mentions ANY competing product, similar tool, or reference platform.

## Research Depth
Research ALL of the following for each competitor:
1. Feature set (from marketing site, docs, changelog)
2. Pricing model (tiers, limits, pricing page)
3. Tech stack (if discoverable from job posts, blog, GitHub)
4. API design (from public API docs)
5. UX patterns (from screenshots, demos, reviews)
6. Review sentiment (G2, Capterra, Reddit, HN)
7. Backend architecture (from engineering blog, conference talks)
8. Recent launches (from changelog, blog, Product Hunt)
9. User complaints (from reviews, forums, support forums)
10. Integration ecosystem (marketplace, partner page)

## Deliverables
1. Feature Gap Matrix: your product vs competitors with coverage %
2. Pricing Comparison Table
3. "Table Stakes" list: features every product in category MUST have
4. "Differentiator Opportunities": gaps in competitors you can exploit
5. "Anti-patterns": features competitors built that users hate

## Integration into Questions
Generate comparison questions:
- "[Competitor] has [X]. Do you want this?"
- "[Competitor] users complain about [Y]. Should we solve this?"
- "[Competitor] charges [Z] for [feature]. Include or premium?"
